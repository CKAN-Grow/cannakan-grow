# Design Bible

## Visual Identity

Grow should feel premium, modern, organic, scientific, evidence-driven, and community-powered.

The interface should balance warmth with rigor. It should feel like a trusted grow intelligence tool, not a generic dashboard and not a decorative marketing site.

## Themes

- Premium
- Modern
- Organic
- Scientific
- Evidence-driven
- Community-powered

## Visual Rules

- Use consistent spacing across pages, cards, controls, and page sections.
- Use rounded cards for grouped information and repeated content.
- Use large hero imagery when a page needs orientation, story, or product context.
- Use soft gradients to add depth without overwhelming the information.
- Use clean typography that supports quick scanning.
- Use green as an accent, not everywhere.
- Put data first.
- Let images support data, not replace it.

## Layout Principles

- Start pages with a clear visual and informational hierarchy.
- Group related information into predictable sections.
- Keep controls close to the data they affect.
- Avoid visual clutter around high-value metrics.
- Make comparison easy by keeping repeated cards structurally consistent.

## Data Presentation

- Show the most useful metric first.
- Pair metrics with plain-language context.
- Indicate confidence when data volume or quality varies.
- Separate verified signals from community signals when both exist.
- Avoid implying certainty when the data is directional.

## Imagery

- Use images to reveal the product, grow state, seed source, community proof, or learning subject.
- Avoid purely atmospheric imagery when users need to inspect or compare.
- Keep imagery secondary to evidence on data-heavy pages.
