# Component Library

## Purpose

The component library defines reusable UI patterns for future Grow development. These components should support consistency, scalability, and demo-ready delivery without forcing large refactors before they are needed.

## Components

### Explorer Hero

Introduces discovery pages such as Explore, Sources, Seeds, Breeders, or Genetics. It should orient the user, state the page purpose, and provide the primary path into the data.

### Report Hero

Introduces detailed report pages such as Source Reports or future Seed Reports. It should summarize identity, trust signals, and key performance metrics.

### Section Header

Labels a major content section with a clear title and optional supporting context. It should make dense pages easier to scan.

### Stat Card

Displays a single high-value metric with a short label and contextual detail. It should be compact, readable, and consistent across dashboards and reports.

### Performance Card

Shows performance-oriented data such as germination rate, seed count, sessions, trends, or comparative outcomes.

### Insight Card

Highlights a meaningful interpretation, recommendation, or pattern discovered from user or community data.

### Gallery Card

Displays community photo or snapshot content with enough context to understand the grow result, source, seed, or session behind it.

### Relationship Card

Represents a grower, source, breeder, or other connected entity and explains why the relationship matters.

### Timeline Card

Shows time-based progress, events, milestones, or grow-stage history in a scannable format.

### Metric Badge

Provides a compact visual label for important attributes such as verified, community-powered, high confidence, coming soon, or demo data.

### Confidence Indicator

Communicates the strength of a signal based on data volume, consistency, freshness, verification, or other trust factors.

### Source Card

Represents a seed source in Explore. It should summarize source identity, performance, confidence, and available report actions.

### Seed Card

Represents a seed or variety in Explore. It should summarize seed identity, source relationships, performance, community evidence, and future report actions.

## Component Principles

- Components should clarify data, not decorate around it.
- Components should be reusable across pillars when the meaning matches.
- Components should preserve current working experiences until replacement is deliberate.
- Components should support progressive disclosure.
- Components should make relationships between Grow data easier to understand.
