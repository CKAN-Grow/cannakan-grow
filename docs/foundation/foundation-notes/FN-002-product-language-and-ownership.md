# Foundation Note FN-002 — Product Language & Ownership

**Status:** Validated  
**Captured:** July 19, 2026  
**Related areas:** Grow Profile, Grow Sessions, Grow Seed Vault, Grow Network, Navigation, UX, Product Language

## Observation

As Grow evolves into a platform, navigation and page experiences serve different purposes.

Navigation exists for speed, clarity, and recognition.

Page titles establish context, reinforce ownership, and strengthen the Grow brand.

Using the same naming convention for both creates unnecessary repetition and makes navigation feel heavier than it needs to be.

## Decision

Navigation should remain short, familiar, and easy to scan.

Major page titles should reinforce the Grow platform through consistent experience naming.

Personal experiences should emphasize ownership using:

**My Grow...**

Shared platform experiences should use:

**Grow...**

## Navigation

Use concise labels:

- Home
- Sessions
- Vault
- Explore
- Community
- Network
- Profile
- Learn

## Page Titles

Personal experiences:

- My Grow Profile
- My Grow Sessions
- My Grow Seed Vault

Shared platform experiences:

- Grow Network
- Grow Community
- Grow Learning
- Grow Explore — subject to future review

## Why It Matters

This creates a consistent product language that:

- reduces navigation clutter
- strengthens the Grow brand
- reinforces ownership where appropriate
- distinguishes personal experiences from shared platform experiences
- scales naturally as Grow expands
- keeps the product aligned with the Grow and Grow Session naming theme

## Foundation Alignment

- Simplicity wins
- Consistent product language
- Reduce cognitive load
- Build once, extend forever
- Personal experiences should feel owned
- Shared experiences should feel connected

## Design Principles

> Brand the experience. Simplify the navigation.

> Personal experiences should feel owned. Shared experiences should feel connected.

## Explicit Non-Goals

This note does not:

- rename navigation now
- redesign any page
- change routes
- change page structure
- alter current labels automatically
- require every page title to begin with Grow
- force “My” onto shared experiences

## Future Implementation Direction

Apply this convention incrementally as pages are redesigned or introduced.

Navigation should remain concise.

Page titles should provide context, ownership, and brand consistency.

Before implementation, confirm each page belongs to one of these categories:

1. Personal experience
2. Shared platform experience
3. Utility or system experience

Avoid mechanical renaming without reviewing the actual experience.
